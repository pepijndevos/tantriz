# Tantriz

A Z-scale (1:220) railroad module system with hexagonal tiles. `hex_tile_box.py`
generates laser-cuttable panels for a hexagonal box module using the
[boxes.py](https://github.com/florianfesti/boxes) generator library.

## Layout

- `hex_tile_box.py` — the generator. Two panel classes, one output file each:
  - `HexTop` → `hex_top.pdf` — hexagonal top deck (×1/tile). Engraves both the curved
    tracks (face pairs 0/2, 0/4, 2/4) and the straight tracks across opposite faces
    (0-3, 1-4, 2-5). All 6 edges use the bracket finger joint (`J`).
  - `WallRail` → `wall_rail.pdf` — one universal chiral side wall (a single part; order ×6
    per tile). Flat `g` finger edge on the right, stepped `G` slot on the left (so all 6 are
    the same part), dowel/clamp holes, cable notch, and a bracket top edge (`j`) with
    rail-catch nibs (`CatchFingerEdge`).
- `*.pdf` — committed output artifacts (regenerated by the script).
- `logo.svg` — project logo.
- `boxes.py/` — cloned upstream boxes library (git-ignored, see Setup). **Not** part of this project's source.
- `.venv/` — local virtualenv (git-ignored).

## Setup

The `boxes` library is installed editable from a local clone.

```sh
git clone https://github.com/florianfesti/boxes.git boxes.py
python3 -m venv .venv
.venv/bin/pip install -e ./boxes.py
```

`boxes.py/` and `.venv/` are git-ignored.

PDF output requires two external tools on PATH (macOS: `brew install ghostscript pstoedit`):
- **Ghostscript** provides `ps2pdf`, which `boxes` uses to convert its PostScript to PDF.
- **pstoedit** must be present for `boxes` to offer `pdf` (and `dxf`/`gcode`/`plt`) as
  `--format` choices at all — see `Formats.getFormats()` in `boxes/formats.py`. Without
  it only `svg`, `svg_Ponoko`, `ps`, `lbrn2` are available and `--format=pdf` errors
  with `invalid choice: 'pdf'`. This is a missing-tool issue, not a removed feature.

## Running

```sh
.venv/bin/python hex_tile_box.py [burn_mm]
```

Writes `hex_top.pdf`, `wall_blind.pdf`, `wall_rail.pdf` to the current directory.

## Geometry notes

Math coordinates: origin at hex center, y-up. boxes.py turtle uses its own heading
convention (see `inward_heading`).

- `S = 415/3 ≈ 138.33 mm` — outer circumradius; `INNER_R ≈ 131.4 mm`; hex side ≈ 131.4 mm.
- `THICKNESS = 6 mm` birch ply; `H_WALL = 50 mm`.
- Two parallel tracks per face at `±12.5 mm` from each face midpoint → **25 mm**
  center-to-center (matches Märklin Z standard spacing).
- Curved tracks: Märklin Z curves R195 (`8591`, inner/C3) and R220 (`8590`, outer/C4),
  engraved across face pairs (0,2)/(0,4)/(2,4) — see `arc_specs()`.
- Straight tracks: the ±12.5 mm pair across opposite faces (0-3)/(1-4)/(2-5) — see
  `straight_specs()`. Both are engraved on the one universal `HexTop`.

### Finger joints

Three finger-joint kinds, all derived from the default `FingerJointSettings`:

- **Corner joints** (wall↔wall, 60°): `gGH` edge chars at `CORNER_ANGLE`. Each wall is
  chiral — `g` (flat) on one vertical, `G` (stepped) on the other — so 6 identical parts
  ring up (the `n % 2` construction from `boxes/generators/regularbox.py`).
- **Bracket top joint** (wall↔top): `j`/`J`, re-spaced to `BRACKET_FINGER=13`,
  `BRACKET_SPACE=12`, `surroundingspaces=1` → **5 tabs** at `0, ±25, ±50 mm`, so the
  notches land exactly on the `±12.5 mm` rail centers. The wall's `j` edge is a
  `CatchFingerEdge` that adds `NIB_W×NIB_H` rail-catch nibs on the notch-facing corners of
  the 3 center tabs (`NIB_TABS`), catching each track base at its edges (≈±6.5, ±18.5 mm).
- Tweak the catch via `NIB_W`, `NIB_H`, `NIB_TABS` (empty map ⇒ flush).

## Conventions

- All dimensions in mm. Burn/kerf compensation via the `burn` arg (default 0.1 mm).
- Geometry constants live in the top block of `hex_tile_box.py`; change them there, not inline.
